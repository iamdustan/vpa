# Project Instructions

- **Workflow**: Do not run the full project build after every code change unless specifically requested or required for a complex verification. Prioritize speed and focus on implementation and surgical updates.
- **Styling**: Prefer Vanilla CSS for maximum flexibility. Avoid TailwindCSS unless explicitly requested (Note: existing components use Tailwind, so maintain consistency in those areas).
- **Backend**: Current preference for simple backends like Google Apps Script for time-to-market.
